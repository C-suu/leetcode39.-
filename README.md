### （1）题目意思与生僻概念解释

**题目意思：**
给定一个没有重复数字的数组 `candidates` 和一个目标值 `target`。要求在这个数组里挑数字，让挑出来的数字之和刚好等于 `target`。
有两个核心规则：
1. **无限重复：** 数组里的同一个数字，可以被无限制地多次挑选（比如选 3 次 `2`）。
2. **组合去重：** `[2, 2, 3]` 和 `[3, 2, 2]` 算作同一种组合，最终的返回结果中只能保留一种。

**核心生僻概念解释：**
* **回溯算法 / 深度优先搜索（DFS / Backtracking）：** 一种通过“试错”来寻找所有可能解的算法。就像走迷宫，先朝着一个方向走到底，如果碰壁了（数字加起来超过了目标值）或者找到出口了（正好等于目标值），就**退回一步**，换一条路继续走，直到走遍所有可能的路径。
* **剪枝（Pruning）：** 在走迷宫（搜索树）的过程中，如果提前预知某条路后面全是死胡同，就直接把这条路“剪断”，不再往下走，以此大幅提升算法的运行速度。
* **组合去重策略（通过 Start Index 控制）：** 为了防止出现 `[2, 3]` 和 `[3, 2]` 这种重复解，代码中引入了参数 `s`（起始索引）。规则是：一旦在本轮选择了某个数字，接下来的探索中，**只能选择该数字及其右边的数字，绝对不能回头选左边的数字**。这就强制保证了所有生成的组合都是按从小到大排序的，从根本上消灭了重复组合。

### （2）代码解题思路解析

这段代码是一个非常经典且经过优化的回溯模板。核心思路是**“先排序，再递归，分三种情况处理”**：

1. **预处理（排序）：** 首先把数组从小到大排好序。这一步是后续能够进行“高效剪枝”的绝对前提。
2. **定义递归任务：** 设定一个探索函数 `find(起始位置, 已经选好的数字列表, 离目标还差多少)`。
3. **分岔探索（三种场景）：** 在当前的有效选择范围内遍历每个数字 `c`：
    * **场景一（正好凑齐）：** 如果 `c` 恰好等于还差的值 `remain`，说明找到了一组有效答案，将其保存。
    * **场景二（还有余额）：** 如果 `c` 小于还差的值 `remain`，说明当前这条路还可以继续往下走。把 `c` 加入列表，更新还差的值，并**继续调用自身**（注意，下次调用的起始位置依然是当前位置 `i`，代表允许重复选择当前数字）。
    * **场景三（彻底超载）：** 如果 `c` 已经大于还差的值 `remain`。因为数组已经从小到大排好序了，既然当前的 `c` 都超标了，它后面更大的数字就更加不可能符合要求。此时直接 `return`，终止当前分支的探索（这就是**剪枝**）。

candidates = sorted(candidates)这个可以写成candidates.sort()吗？两个函数的区别是啥？

这两个函数虽然都能完成排序，但在底层逻辑和内存开销上有显著的区别。总结来说就是：**一个是“原地整容”，一个是“克隆后整容”。**

直接写一句 **`candidates.sort()`**，既干净利落，又省了内存

### 核心区别对比

| 特性 | `candidates.sort()` (列表方法) | `sorted(candidates)` (内置函数) |
| :--- | :--- | :--- |
| **执行方式** | **原地修改 (In-place)**。直接把原来的列表重新排列。 | **生成新对象**。保持原列表不变，克隆一份排好序的新列表。 |
| **返回值** | 返回 `None`。 | 返回一个新的、排好序的列表。 |
| **内存开销** | **极小**。因为没有创建新列表。 | **较大**。需要为新生成的列表开辟一块新的内存空间。 |
| **适用范围** | **只能用于列表 (list)**。它是列表的专属方法。 | **适用于所有可迭代对象** (如元组 tuple、字符串 str、字典 dict 的键等)。 |

### 具体代码演示

**1. 使用 `candidates.sort()`**
```python
nums = [3, 1, 2]
res = nums.sort()  

print(nums)  # 输出: [1, 2, 3] (原列表被修改了)
print(res)   # 输出: None (没有返回值)
```
*注意：千万不能写成 `nums = nums.sort()`，这会让 `nums` 变成 `None`！*

**2. 使用 `sorted(candidates)`**
```python
nums = [3, 1, 2]
res = sorted(nums) 

print(nums)  # 输出: [3, 1, 2] (原列表毫发无损)
print(res)   # 输出: [1, 2, 3] (生成了一个全新的列表)
```

### （3）带有注释的代码框

```python
class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        # 1. 预处理：对数组进行升序排序，为后续的“提前剪枝”做准备
        candidates = sorted(candidates)
        # 2. 初始化全局结果列表，用于存放所有符合条件的组合
        ans = []
        # 3. 定义深度优先搜索（回溯）函数
        # s: 当前允许选择的起始索引（用于去重）
        # use: 当前路径上已经选择的数字列表
        # remain: 距离目标和还剩下的数值
        def find(s, use, remain):
            # 从起始索引 s 开始，向后遍历候选数字
            for i in range(s, len(candidates)):
                # 场景一：当前数字恰好填满剩余值，找到一个有效组合
                if candidates[i] == remain:
                    ans.append(use + [candidates[i]])
                # 场景二：当前数字小于剩余值，路径有效，继续深入探索
                # 传入的下一轮起始索引依然是 i，表示当前数字可以被无限制重复选取
                if candidates[i] < remain:
                    find(i, use + [candidates[i]], remain - candidates[i])
                # 场景三：当前数字已经大于剩余值，直接触发剪枝
                # 因为数组已排序，当前数字超标，其右侧的数字必然全部超标，直接结束本层循环
                if candidates[i] > remain:
                    return
        # 4. 触发首次探索：从索引 0 开始，已选列表为空，剩余值为初始 target
        find(0, [], target)
        return ans
```

### （4）每一行代码详细中文解释

1.  `class Solution:`
    * **解释：** 定义算法平台标准的题解类。
2.  `def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:`
    * **解释：** 定义主方法，接收候选数组 `candidates` 和目标整数 `target`，返回一个由列表构成的列表（二维列表）。
3.  `candidates = sorted(candidates)`
    * **解释：** 调用内置排序功能，将传入的数组重新按照从小到大的顺序排列，并覆盖原变量。这是优化算法执行效率的核心动作。
4.  `ans = []`
    * **解释：** 初始化一个空列表，用来收集所有满足总和等于 `target` 的有效数字组合。
5.  `def find(s, use, remain):`
    * **解释：** 在主方法内部定义一个辅助的递归函数（闭包）。接收三个参数：`s` 代表当前可以从数组的哪个下标开始挑数字；`use` 代表沿着当前路径一路走来已经挑进篮子里的数字；`remain` 代表要凑齐目标总和还欠缺的数值。
6.  `for i in range(s, len(candidates)):`
    * **解释：** 开启一个循环。从允许的最小索引 `s` 开始，一直遍历到数组的最末尾。控制起始位置 `s` 可以确保不会走回头路，避免生成重复的组合。
7.  `c = candidates[i]`
    * **解释：** 将当前遍历到的候选数字提取出来，赋值给临时变量 `c`，方便后续进行比较。
8.  `if c == remain:`
    * **解释：** 判断当前提取出的数字 `c` 是否刚好等于还需要凑齐的数值 `remain`。
9.  `ans.append(use + [c])`
    * **解释：** 如果条件成立，说明这条路径走通了。将之前积累的数字列表 `use` 与当前的数字 `c` 合并成一个完整的新列表，并追加到全局结果 `ans` 中保存下来。
10. `if c < remain:`
    * **解释：** 检查当前数字 `c` 是否小于还需凑齐的数值。这意味篮子还没装满，还可以继续往里放。
11. `find(i, use + [c], remain - c)`
    * **解释：** 既然没装满，就递归调用 `find` 函数向更深层探索。重点在于第一个参数传的是 `i` 而不是 `i + 1`，这赋予了算法“无限次重复选取同一个数字”的能力。同时更新已选列表（加入 `c`）和剩余数值（减去 `c`）。
12. `if c > remain:`
    * **解释：** 检查当前数字 `c` 是否已经大于所需的剩余数值。说明如果把这个数字放进篮子，总和就会爆炸。
13. `return`
    * **解释：** 既然当前的数字 `c` 已经让总和超标，由于数组早就排好序了，后面遍历到的数字只会比 `c` 更大，更加会超标。因此无需继续 `for` 循环，直接执行 `return` 退出当前函数调用，退回到上一层节点继续探索其他路径（也就是剪枝操作）。
14. `find(0, [], target)`
    * **解释：** 辅助函数定义完毕。正式拉开算法序幕，从下标 `0` 开始全盘扫描，初始携带的篮子为空 `[]`，需要凑齐的目标数值为最初的 `target`。
15. `return ans`
    * **解释：** 当最初的那次 `find` 调用彻底执行完毕时，说明搜索树的所有分支都已经探索和修剪完毕，将收集到的所有结果 `ans` 返回。

### （5）具体数值算例与变量变化过程

**设定算例：** 输入 `candidates = [2, 3, 6, 7]`，`target = 7`。
（数组已被排序）

以下为回溯探索树的部分核心推演过程：

| 探索层级 / 动作 | `s` (当前起始索引) | `i` (循环指向下标) | `c` (当前候选数字) | `use` (当前已选列表) | `remain` (剩余需要凑齐的数值) | 逻辑判定与代码流向 | 对应代码行 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **初始调用** | `0` | - | - | `[]` | `7` | 开始搜寻全数组 | `find(0, [], 7)` |
| ├── **深入一层** | `0` | `0` | **2** | `[]` | `7` | `2 < 7`，放入篮子，继续深入 | `if c < remain: find(i, ...)` |
| │   ├── **深入二层** | `0` | `0` | **2** | `[2]` | `5` | `2 < 5`，放入篮子，继续深入 | `if c < remain: find(i, ...)` |
| │   │   ├── **深入三层** | `0` | `0` | **2** | `[2, 2]` | `3` | `2 < 3`，放入篮子，继续深入 | `if c < remain: find(i, ...)` |
| │   │   │   ├── **深入四层** | `0` | `0` | **2** | `[2, 2, 2]` | `1` | `2 > 1`，爆额！**触发剪枝，退出该层** | `if c > remain: return` |
| │   │   │   └── **回退后并进**| `0` | `1` | **3** | `[2, 2]` | `3` | `3 == 3`，**凑齐！加入结果集** `ans = [[2, 2, 3]]` | `if c == remain: ans.append(...)` |
| │   │   │   └── **继续并进** | `0` | `2` | **6** | `[2, 2]` | `3` | `6 > 3`，爆额！**触发剪枝，退出该层** | `if c > remain: return` |
| │   │   └── **回退后并进**| `0` | `1` | **3** | `[2]` | `5` | `3 < 5`，放入篮子，继续深入... | `if c < remain: find(i, ...)` |
| │   │       └── ... | ... | ... | ... | ... | ... | *(以此类推，尝试 [2,3,2], [2,3,3] 等均因去重规则或爆额被修剪或排除)* | ... |
| ├── **回退至首层** | `0` | `1` | **3** | `[]` | `7` | `3 < 7`，放入篮子，继续深入 | `if c < remain: find(i, ...)` |
| │   └── ... | ... | ... | ... | ... | ... | *(以此类推)* | ... |
| ├── **回退至首层** | `0` | `2` | **6** | `[]` | `7` | `6 < 7`，放入篮子，继续深入 | `if c < remain: find(i, ...)` |
| │   └── ... | ... | ... | ... | ... | ... | *(发现 6+2>7, 6+3>7，剪枝回退)* | ... |
| └── **回退至首层** | `0` | `3` | **7** | `[]` | `7` | `7 == 7`，**凑齐！加入结果集** `ans = [[2, 2, 3], [7]]` | `if c == remain: ans.append(...)` |

最终结果集为 `[[2, 2, 3], [7]]`。
